## 1. Overview

### 1.1 Project Summary

Мобильный сайт для «К Студии» — mid-premium салона красоты в Астане, предоставляющего полный спектр услуг: парикмахерские (стрижки, окрашивание, укладки), маникюр и педикюр, макияж и другие beauty-услуги. Сайт должен транслировать уровень и атмосферу студии, снимать барьер первого звонка через онлайн-запись 24/7, и убеждать потенциального клиента через галерею работ и профили мастеров — ещё до первого визита.

### 1.2 Business Objectives

- **Увеличить поток новых клиентов** за счёт органического поиска в Google и Яндекс по запросам «[услуга] Астана» — цель: +30% новых клиентов в первые 6 месяцев после запуска
- **Перевести запись в онлайн** — устранить зависимость от телефонных звонков и мессенджеров; цель: 50% записей через сайт в течение 3 месяцев после запуска
- **Установить mid-premium позиционирование** через качество визуала, прозрачность цен и профессиональные профили мастеров, чтобы обосновать ценовой уровень выше среднего по рынку Астаны
- **Стать точкой агрегации социального доказательства** — собирать и демонстрировать отзывы с Google, 2GIS, Instagram
- **Увеличить LTV клиентов** — через напоминания о записи, прозрачный прайс и удобство повторного бронирования

### 1.3 Success Metrics

- **Online booking conversion rate > 3%** (посетители сайта → завершённые записи)
- **Мобильный bounce rate < 45%** (70%+ трафика будет с мобильных)
- **LCP < 2.5 сек** на мобильных устройствах (4G-сеть)
- **Доля онлайн-записей > 50%** от общего числа записей через 3 месяца
- **Органический трафик > 500 уникальных посетителей/месяц** через 6 месяцев после запуска

## 2. Target Audience

### 2.1 Primary Persona — Айгерим (29, работающая женщина)

- **Демография:** Женщины 25–38 лет, Астана, средний и выше-среднего доход (от 300 000 ₸/месяц)
- **Образ жизни:** Активная карьера, ценит своё время, следит за трендами через Instagram и Pinterest, часто занята в рабочие часы
- **Мотивации:** Хочет выглядеть ухоженно, чувствовать уверенность; ценит качество и результат выше цены; не хочет объяснять по телефону, что именно ей нужно
- **Боли:** Не может позвонить в рабочее время; не знает, кому именно доверить волосы; боится «как-то не то» после окрашивания; хочет видеть работы мастера заранее
- **Поисковое поведение:** «Окрашивание волос Астана», «маникюр недалеко», «[название студии] отзывы», ищет по Instagram-аккаунту
- **Тайминг визита на сайт:** Вечером (с 20:00 до 23:00), в обеденный перерыв (12:00–13:00), в выходные

### 2.2 Secondary Persona — Камила (40, руководитель / предприниматель)

- **Демография:** Женщины 38–52 лет, устоявшийся доход, ценит постоянство и экспертизу
- **Мотивации:** Хочет своего мастера, которому доверяет; ценит безупречный результат и минимум объяснений; готова платить за качество
- **Боли:** Устала от поиска нового мастера; хочет видеть опыт специалиста до записи
- **Поведение:** Изучает сайт основательно; читает биографии мастеров; смотрит галерею работ по своему типу волос/стилю

### 2.3 User Goals (Jobs-to-Be-Done)

1. **Найти подходящего мастера** — увидеть его работы, специализацию, опыт до принятия решения
2. **Записаться в удобное время без звонка** — 24/7, самостоятельно, с подтверждением на WhatsApp/email
3. **Узнать точную цену** — не «от X», а реальный диапазон по конкретной услуге
4. **Посмотреть примеры работ** — до/после, похожие на свой запрос
5. **Убедиться в надёжности студии** — отзывы реальных клиентов, рейтинг, соцсети

## 3. Scope

### 3.1 In Scope (MVP — v1)

- Адаптивный mobile-first сайт (мобильный + планшет + десктоп)
- Главная страница с hero-секцией, услугами, галереей, командой, отзывами
- Страница услуг с категориями, описаниями и прайсом
- Галерея работ (до/после, фильтрация по типу услуги)
- Страница команды с профилями мастеров
- Страница «О студии»
- Страница контактов с картой (2GIS + Google Maps)
- Онлайн-запись (интеграция с Yclients или DIKIDI)
- FAQ
- Базовый Local SEO (schema, мета-теги, Google Business Profile)
- HTTPS, cookie consent, политика конфиденциальности
- Аналитика (GA4 + Яндекс.Метрика)

### 3.2 Out of Scope (v2+)

- Личный кабинет клиента с историей записей
- Программа лояльности / бонусы
- E-commerce (продажа продуктов)
- Видеоконтент на hero (для старта — статичный hero)
- AI-виртуальная примерка
- Мобильное приложение
- Многоязычность (рус + каз + eng) — v2
- Чат-бот / онлайн-консультант
- Блог (запустить через 3–6 месяцев после основного сайта)
- Push-уведомления

### 4.3 User Flows

**Flow 1: Новый клиент → Запись через галерею**
1. Поиск в Google «окрашивание волос Астана» → посадочная страница `/uslugi/volosy`
2. Смотрит описание услуги и цены
3. Переходит в галерею → видит похожие работы
4. Нажимает «Записаться» → выбирает услугу → выбирает мастера
5. Выбирает дату и время → вводит имя + телефон
6. Получает подтверждение в WhatsApp или SMS

**Flow 2: Клиент по рекомендации → Запись к конкретному мастеру**
1. Подруга посоветовала Айгуль из К Студии → заходит на сайт
2. Открывает `/masters` → находит профиль Айгуль
3. Смотрит её работы в мини-галерее
4. Нажимает «Записаться к Айгуль» → сразу переходит к выбору даты/времени
5. Завершает запись → получает подтверждение

**Flow 3: Постоянный клиент → Повторная запись**
1. Заходит на сайт с закладки
2. Нажимает «Записаться» в header
3. Система booking предлагает последние услуги → выбирает
4. Записывается за 2 клика

## 5. Page Requirements

### Главная страница `/`

**Цель:** Создать первое впечатление уровня «я хочу именно сюда», конвертировать в запись или просмотр галереи

**Структура и контент:**
1. **Hero** — профессиональное фото трансформации (до/после) или атмосферное фото интерьера/мастера в работе. H1: позиционирование студии. Подзаголовок: 1 строка УТП. CTA: «Записаться онлайн» (primary) + «Смотреть работы» (secondary)
2. **Trust bar** — 3–4 цифры: «___ клиентов», «___ лет на рынке», **«Рейтинг 4.9★ (68+ отзывов)»**, «___ мастеров»
3. **Услуги** — 4–6 карточек (Стрижка, Окрашивание, Маникюр, Педикюр, Макияж, + другое). Каждая: иконка/фото + название + цена «от...» + кнопка
4. **О студии** — 2–3 предложения о философии + фото интерьера или команды
5. **Галерея** — 6–9 лучших работ в masonry grid, кнопка «Смотреть все работы»
6. **Мастера** — 3–4 карточки мастеров: фото + имя + специализация + «Записаться»
7. **Отзывы** — карусель из 4–6 отзывов (Google / 2GIS + реальные фото клиентов)
8. **Booking CTA** — полноширинный блок с контрастным фоном: «Запишитесь онлайн — это займёт 2 минуты» + большая кнопка
9. **Footer**

**CTA:** «Записаться онлайн» — sticky на mobile (фиксированная кнопка внизу экрана)

## 7. Design Requirements

### 7.1 Brand & Positioning

**Сегмент:** Mid-Premium — выше массового рынка Астаны, ниже luxury. Клиент платит за качество, экспертизу и внимание к деталям.

**Эмоция бренда:** Уверенная элегантность. Студия, которая знает себе цену и оправдывает её.

**Характер бренда:** Профессиональный, тёплый, современный, внимательный, честный

**Ключевые слова стиля:** Clean. Elevated. Personal.

### 7.2 Visual Direction

**Цветовая палитра** (направление «Quiet Luxury» для mid-premium):

| Роль | Цвет | HEX | Описание |
|------|------|-----|----------|
| Primary | Тёплый белый / слоновая кость | `#F8F5F1` | Фон, воздух, пространство |
| Secondary | Глубокий тёмно-коричневый | `#2C1F14` | Заголовки, текст, якорный цвет |
| Accent | Тёплое золото / шампань | `#C9A96E` | CTA-кнопки, акценты, иконки |
| Neutral | Светло-бежевый | `#E8E0D5` | Фон секций, карточки |
| Text | Тёмно-серый | `#3A3A3A` | Основной текст |

> ⚠️ **Open Question #1:** Подтвердить палитру с клиентом — возможна корректировка под существующий фирменный стиль «К Студии»

**Типографика:**
- **Заголовки (H1–H3):** Cormorant Garamond или Playfair Display — элегантный serif, ощущение люкса
- **Подзаголовки / UI:** Montserrat Medium — современный, читаемый
- **Body text:** Montserrat Regular, 16px минимум — чёткость на мобильном

**Стиль изображений:**
- Профессиональная съёмка: тёплое освещение, нейтральный фон
- Единая цветовая обработка всех фото (warm/beige tone correction)
- Реальные работы мастеров — никаких стоков
- Атмосферные кадры интерьера студии
- Портреты мастеров: естественный свет, approachable expression

### 7.3 UI Patterns

- **Карточки:** Мягкие тени, скруглённые углы 8–12px, hover: лёгкое поднятие (translateY -4px)
- **Кнопки (primary):** Заливка цветом `#C9A96E`, белый текст, радиус 6px, hover: champ → dark brown
- **Кнопки (secondary/ghost):** Прозрачный фон, граница `#C9A96E`, текст `#2C1F14`
- **Навигация:** Прозрачный header на hero → белый/тёмный после scroll; на мобильном — drawer-меню
- **Галерея:** Masonry grid без gap (или минимальный), клик → плавный lightbox fade-in
- **Анимации:** Fade-in + slight translateY при появлении элементов в viewport; parallax на hero — опционально
- **Формы:** Лаконичные поля с floating labels, без лишних границ; валидация inline

## 8. Non-Functional Requirements

### 8.1 Performance

| Метрика | Цель | Инструмент проверки |
|---------|------|---------------------|
| **LCP** (Largest Contentful Paint) | < 2.5 сек | PageSpeed Insights, Lighthouse |
| **INP** (Interaction to Next Paint) | < 200 мс | CrUX |
| **CLS** (Cumulative Layout Shift) | < 0.1 | Lighthouse |
| **TTI** (Time to Interactive) | < 4.0 сек | Lighthouse |

**Технические меры:**
- Все изображения в формате **WebP** с PNG/JPG fallback
- Lazy loading для изображений за пределами первого viewport
- Максимум **200 КБ** на изображение в мобильном viewport
- CDN для статичных ресурсов (Cloudflare или аналог)
- Минификация и сжатие CSS/JS
- Ограничить сторонние скрипты (подключать GA4, Метрику async)

### 8.2 Mobile & Accessibility

- **Mobile-first разработка** — сначала mobile layout, затем tablet/desktop
- **Минимум tap target:** 44×44px для всех интерактивных элементов
- **WCAG 2.1 Level AA:**
  - Контраст текста ≥ 4.5:1 (body), ≥ 3:1 (заголовки крупные)
  - Alt-текст на всех изображениях
  - Доступность с клавиатуры (focus styles, порядок табуляции)
  - Aria-labels на иконках без текста
- **Шрифт:** Минимум 16px для body text на мобильном
- **Горизонтальный скролл:** Запрещён
- **Тестирование:** Chrome DevTools mobile emulation + реальный iOS Safari + Android Chrome

### 8.3 SEO

**URL структура:** Кириллица или транслит — выбрать одно, не мешать. Рекомендация: `/uslugi/strizhka-astana`, `/uslugi/okrashivanie-volos-astana`

**Meta tags (шаблон):**
- Title: `[Услуга] в Астане | К Студия — запись онлайн` (≤ 60 символов)
- Description: `[Услуга] в К Студии, Астана. Опытные мастера, прозрачные цены, запись 24/7 онлайн. [Рейтинг/цифра доверия]` (≤ 160 символов)
- Open Graph: title, description, image (1200×630px) для каждой страницы

**Schema Markup:**
- `LocalBusiness` (+ `BeautySalon`) на главной: name=«К студия», address=«Кабанбай батыр проспект, 58Б, Астана», phone=«+7-705-117-XXXX», hours=«Mo-Su 10:00-21:00», geo, url
- `Service` для каждой категории услуг
- `Person` для каждого мастера
- `Review` + `AggregateRating` для отзывов

**Local SEO (ключевые действия):**
- Google Business Profile: заполнить полностью, добавить фото, включить кнопку «Записаться»
- **2GIS** (критично для Казахстана): создать/обновить профиль, добавить все данные
- **Яндекс.Карты:** зарегистрировать организацию
- NAP consistency (Название / Адрес / Телефон) — идентично везде
- Ключевые страницы под запросы: «стрижка Астана», «окрашивание волос Астана», «маникюр Астана»

### 8.4 Security & Privacy

- **HTTPS** обязателен (SSL — Let's Encrypt или платный через хостинг)
- **Cookie consent banner** согласно Закону РК о персональных данных и GDPR (если есть клиенты из ЕС)
- **Политика конфиденциальности** — страница `/privacy`, на русском языке
- **Защита форм:** reCAPTCHA v3 или honeypot поле
- **Booking система** (Yclients/DIKIDI) обрабатывает чувствительные данные — не хранить телефоны клиентов на собственном сервере
- **Резервное копирование:** Ежедневный бэкап на хостинге или через GitHub
